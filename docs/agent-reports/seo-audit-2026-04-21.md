## P0 — KRİTİK PROBLEMLƏR (dərhal, bu həftə)

### P0-1. HTML səhifələri heç vaxt cache olunmur (`no-store`)

**Problem:** Bütün HTML cavabları:
```
cache-control: private, no-cache, no-store, max-age=0, must-revalidate
x-vercel-cache: MISS
```

Test edilmiş 6 səhifə (`/`, `/en`, `/tools/pdf-merge`, `/en/tools/pdf-merge`, `/blog`, `/info`) — hamısı MISS. Bu o deməkdir ki:
1. Hər tələb Vercel function-ı işə salır (876 SSG səhifə olsa da, edge-də cache olunmur)
2. Googlebot crawl budget-i boş yerə xərclənir
3. TTFB yüksəkdir (fra1 region-dan AZ user-inə ~200-400ms əlavə)
4. Vercel edge CDN faktiki istifadə olunmur

**Niyə vacibdir:** Google INP və TTFB Core Web Vitals metric-lərinə birbaşa təsir. 876 URL-lik SEO sayt üçün — kritik. Həm də "crawl-ed but not indexed" status yaranır.

**Səbəb (ehtimalla):** `next-intl` middleware hər istəkdə `set-cookie: NEXT_LOCALE=az` qoyur — `private, no-cache` davranışını tetikləyir. Həmçinin `setRequestLocale` dinamik rendering səbəb ola bilər.

**Həll:**
```typescript
// next.config.js — async headers() içində əlavə et:
{
  source: '/((?!_next|api|_vercel).*)',
  headers: [
    { key: 'Cache-Control', value: 'public, s-maxage=3600, stale-while-revalidate=86400' },
    { key: 'CDN-Cache-Control', value: 'max-age=3600, stale-while-revalidate=86400' },
    { key: 'Vercel-CDN-Cache-Control', value: 'max-age=86400' },
  ],
},
```

Middleware-də `NEXT_LOCALE` cookie-ni şərti set et (`createIntlMiddleware({ localeDetection: false })` və ya manual cookie set).

**Yoxlama:**
```bash
curl -sLI "https://vaxtimyoxdu.com/" | grep -iE "cache-control|x-vercel-cache"
# Gözlənən: x-vercel-cache: HIT, cache-control: public, s-maxage=3600
```

**İş yükü:** 3-6 saat
**Təsir:** TTFB −50%, Core Web Vitals +15-30 ranking pozisiyası, crawl budget 3-5x artır

### P0-2. News article-larda heç bir `<img>` yoxdur

**Problem:** `/info/nba-play-in-final-gunu-hornets-magic-warriors-suns` səhifəsində `<img>` tag sayı **sıfırdır**. NewsArticle JSON-LD `image` sahəsi OG URL-ə bağlıdır, amma səhifənin real content-ində heç bir şəkil yoxdur.

**Niyə vacibdir:**
1. **Google News Top Stories** daxil olma bloklanır (artikl body-sində optimised hero image tələbi var)
2. **E-E-A-T** — şəkilsiz xəbər = low-effort content
3. **Google Images** trafiki sıfırdır
4. **SERP CTR** — thumbnail olmayan xəbərlər 40% az klik alır

**Həll (iki mərhələli):**

**Mərhələ 1 (2 saat):** Hər xəbərdə API OG image-i hero kimi göstər:
```tsx
// src/app/[locale]/info/[slug]/page.tsx
<Image
  src={`/api/og?title=${encodeURIComponent(article.title)}&type=news`}
  alt={article.title}
  width={1200}
  height={630}
  priority
  className="w-full rounded-lg mb-6"
/>
```

**Mərhələ 2 (1-2 gün):** `news-articles.ts`-ə `heroImage?: string` sahəsi əlavə et, top 20 xəbər üçün real editorial image (unsplash / öz dizayn). `/public/news/<slug>.webp` altında 1200×675 (16:9), next/image ilə lazy load.

**İş yükü:** Mərhələ 1 = 2 saat, Mərhələ 2 = 1-2 gün (editorial workflow)
**Təsir:** Google News Top Stories-ə daxil olmaq imkanı, +30-50% SERP CTR

## P1 — YÜKSƏK PRIORITET (1-2 həftə)

### P1-1. HowTo schema `HowToStep.name` generic ("Step 1", "Step 2")

**Problem:** `src/lib/utils/seo/json-ld.ts:90` — rich content variantında hər step-in `name` sahəsi `Step ${index + 1}` generic formatındadır. SERP-də HowTo rich result-da "Step 1" kimi göstərir — meaningless.

**Həll:**
```typescript
step: richContent.howToUse.map((stepText, index) => ({
  '@type': 'HowToStep',
  position: index + 1,
  name: stepText.split(/[.!?]/)[0].trim().slice(0, 60),
  text: stepText,
  url: `${url}#step-${index + 1}`,
}))
```

`ToolContentSection.tsx`-də hər step-ə `id="step-N"` əlavə et.

**İş yükü:** 2 saat
**Təsir:** HowTo rich result eligibility, SERP-də meaningful step preview

### P1-2. 95 alət (111-dən) generic İngilis metaTitle template istifadə edir

**Problem:** `src/config/tools/*` analizindən: 111 tool-dan yalnız 16-nın localized `metaTitle` və 18-nin `metaDescription` override-u var. **Nümunə problem:**
- `/tr/tools/json-formatter` title: `"JSON Formatter — Free, No Upload Required | Vaxtim Yoxdu"` (TR saytında İngilis!)
- `/ru/tools/uuid-generator` eyni problem

```typescript
const titleSuffixByLocale = {

return generateToolMetadata(tool, {
```

### P2-4. Sitemap 876 URL, amma index sitemap yoxdur

**Problem:** Hal-hazırda `sitemap.xml` = monolith. News sitemap `<news:news>` schema istifadə etmir (Google News üçün kritik).

**Həll:** Multi-sitemap index:
```
sitemap.xml (index) →
  /sitemap-static.xml    (static, weekly)
  /sitemap-tools.xml     (444 URL, monthly)
  /sitemap-news.xml      (son 2 gün, news:news schema, daily)
  /sitemap-blog.xml      (blog)
  /sitemap-info-old.xml  (2 gündən köhnə xəbərlər)
```

News sitemap example:
```xml
<url>
  <loc>https://vaxtimyoxdu.com/info/article-slug</loc>
  <news:news>
    <news:publication>
      <news:name>Vaxtim Yoxdu</news:name>
      <news:language>az</news:language>
    </news:publication>
    <news:publication_date>2026-04-18T12:00:00Z</news:publication_date>
    <news:title>Article title</news:title>
  </news:news>
</url>
```

Next.js 15-də `src/app/sitemap-news.xml/route.ts`.

**İş yükü:** 1 gün
**Təsir:** Google News daxil olma, crawl efficiency

## Nə Yaxşıdır (Dəyişdirmək Lazım Deyil)

Auditdə həmçinin mükəmməl işlənmiş sahələri qeyd etmək vacibdir:

1. **hreflang implementasiyası** — 3 yerdə uyğun (HTML, HTTP header, sitemap). `x-default` düzgün AZ-a yönəldilir. `url.ts:25` default locale trailing slash logic qüsursuzdur.

4. **Locale-specific articles** — cross-locale hreflang emit etmir (ağıllı qərar, 404 qarşısını alır).
5. **Security headers** — HSTS preload, X-Frame-Options DENY, Permissions-Policy, CSP hardened.
6. **Redirect stratejisi** — vaxtimyoxdur.com → vaxtimyoxdu.com 301, `/news` və `/xeberler` → `/info` 301.
7. **Breadcrumbs** — həm visual, həm JSON-LD.
8. **Robots.txt** — Googlebot/Bingbot explicit rules, `Sitemap` və `Host` direktivləri.
9. **Fixed toolsLastModified** — crawl budget-i qoruyur.
10. **PWA + OG** — manifest, apple-touch-icon, dynamic OG image generator.
